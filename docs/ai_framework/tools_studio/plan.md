 Tools Studio — Phase 1: Tool Registry, Catalog & Inspector                                                                                                                                                                  │
│                                                                                                                                                                                                                             │
│ Context                                                                                                                                                                                                                     │
│                                                                                                                                                                                                                             │
│ The framework has rich tool-like functionality spread across 4 services (Tavily search, Email/Resend, Memory/MongoDB cache, RAG/Weaviate) plus LLM provider methods, but none are registered as discoverable tools. The     │
│ existing /api/v1/tools endpoint is a stub returning []. The existing ToolsPage renders an empty ToolList. Tools Studio replaces the empty Tools page with a discovery-based studio (mirroring Agent Studio) that registers  │
│ all 16 framework tools in a static registry, provides a catalog grid with category filtering, and offers a full-page inspector with schema and "Try It" execution.                                                          │
│                                                                                                                                                                                                                             │
│ Key Design Decisions                                                                                                                                                                                                        │
│                                                                                                                                                                                                                             │
│ 1. Static tool registry — A _TOOL_REGISTRY list in the endpoint file defines all tools with metadata (mirrors _AGENT_FACTORIES pattern). No DB needed.                                                                      │
│ 2. Service-backed execution — Each registry entry maps to a service class + method. The try endpoint instantiates the service and calls the method with a 60s timeout.                                                      │
│ 3. Reuse existing /tools route — Replace the empty ToolsPage content. Add /tools/:toolId inspector route (URL-encode the / in tool IDs as --, e.g. search--tavily_search_topic).                                            │
│ 4. Category filtering — 5 categories: search, email, cache, rag, llm.                                                                                                                                                       │
│ 5. Rename sidebar — "Tools" becomes "Tools Studio" (Wrench icon stays).                                                                                                                                                     │
│                                                                                                                                                                                                                             │
