  What Phase 2 Enables
  --------------------                                                                                                                                                                                                         
                                                                                                                                                                                                                               
  The models and repositories created in Phase 2 form the data persistence layer for the entire newsletter platform. The Newsletter model stores generated newsletter content (markdown, HTML, plain text), tracks the         
  generation workflow status (draft → generating → pending_review → ready → sent), and preserves the outputs from each agent (research data, writing data, mindmap). This allows users to review, edit, and approve newsletters
   at each HITL checkpoint before sending. The Subscriber model maintains the email list with per-subscriber preferences (topics of interest, preferred tone, frequency) and engagement metrics (open rates, click rates),     
  enabling personalized newsletter generation and analytics tracking.                                                                                                                                                          
                                                                                                                                                                                                                               
  The Campaign model ties everything together for delivery - it links a newsletter to a subset of subscribers (filtered by tags/groups), handles scheduling, and tracks delivery analytics (delivered, opened, clicked,        
  bounced). The Template model provides reusable HTML/text layouts with variable placeholders, so users can maintain consistent branding across newsletters. The repositories provide the database operations (create, read,   
  update, delete) plus specialized methods like bulk_create for importing subscribers, find_scheduled for the scheduler to find campaigns ready to send, and update_engagement to track email opens/clicks. These will be used 
  by the agents (Phases 6-9), the LangGraph orchestrator (Phase 10), and the API endpoints (Phase 12) to persist and retrieve data throughout the newsletter generation and delivery workflow.
